# WLEdit - BrickLink Studio 用 Wanted List エディター
BrickLinkで扱っているXML形式パーツリストに簡単な操作ができる、発注関連作業円滑化のために作成したPythonスクリプトです。
## 用例
- パーツを少し多めに頼んで店舗側のミスに備えたいとき：
  - パーツリストにパーツを全種類2個ずつ追加  
    `python wledit.py [old_list.xml] * * 2 -o [new_list.xml]`
  - パーツリストのパーツを4%増加し切り上げ  
    `python wledit.py [old_list.xml] -m 1.04 -o [new_list.xml]`
- 発注したパーツに何らかのエラーで不足が生じていないか確認したいとき：
  - 各ショッピングカートの内容を個々のリストに出力して作品のパーツリストと照会  
    `python wledit.py [order1.xml] [order2.xml] [order3.xml] -s [model.xml] -p`
  - 一定期間の発注内容を一括ダウンロードしたファイルと照会  
    `python wledit.py [orderxml.txt] -s [model.xml] -p`
## 使い方
まずはBrickLinkから最新の色定義を取得しましょう。  
使い方は単に`python getcolor.py`を実行するだけです。  
これで色の名称とIDがbricklink.jsonに保存されます。

次にWLEditの使用法です。  
使い方は
- `python wledit.py`を実行して入力欄を呼び出す方法と
- `python wledit.py add list1.xml list2.xml out list3.xml`のようにコマンドを先に指定して直接実行する方法があります。

前者の場合`quit`もしくは`-q`を入力しないとプログラムが終了しません。  
わからなくなったら`help`もしくは`-h`を入力しましょう。

まずはXMLリストを読み込んでみます。  
StudioからパーツリストをXML形式で出力するにはFile>Export As>Export as WL xml...を選択します。  
操作したいXMLファイルとwledit.pyが同じフォルダ内に置かれているとファイルの呼び出しが楽です。  
読み込みには`add`もしくは`-a`コマンドを使用しますが、冒頭における場合のみにつき省略可能です。  
`python wledit.py`を実行したのち、ファイル名を入力すると読み込まれます。（拡張子の.xmlは省略可能です）  
`python wledit.py list`のようにコマンドを実行時に指定してしまうとリストが表示されません。  
`print`もしくは`-p`を末尾につけて表示させます。  
複数のファイルを同時に指定するとそれらの内容が合計されます。

ファイルの内容を差し引くには`sub`もしくは`-s`コマンドを使用します。  
また追加・削減したいパーツを指定することも可能です。  
そのためには`-[a|s] [part_id] [color_id] [quantity]`のように、型番・色番・個数の順にパーツを指定していきます。  
Wanted list同様、一度に複数種のパーツを追加・削減することができます。  
また型番・色番には`*`のワイルドカードが使用できます。  
`-a * 1 1`と入力するとパーツリストのうち、白いパーツを1個ずつ増やすことができます。

編集した内容のリストを保存するには`out`もしくは`-o`コマンドを使用します。  
読み込み時同様.xmlは省略可能で、ファイル名を指定しない場合は標準でout.xmlになります。  
間違えて既存のファイルを上書きしないよう注意しましょう。
